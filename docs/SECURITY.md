## Reporting a vulnerability

We take security seriously. If you discover a security vulnerability in Mintax, please **do not** open a public issue.

Instead, email **security@mindtris.com** with:

- A description of the issue
- Steps to reproduce
- Affected version(s)
- Your contact information (for follow-up questions)

## What to expect

| Stage | Timeline |
|-------|----------|
| Initial response | Within 48 hours |
| Triage and validation | Within 5 business days |
| Fix development | Depends on severity |
| Coordinated disclosure | Aligned with reporter |

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.5.x   | Yes       |
| < 0.5   | No        |

## Responsible disclosure

We follow responsible disclosure practices:

1. You report the issue privately
2. We confirm and work on a fix
3. We release the fix
4. We credit you in the release notes (unless you prefer anonymity)

Thank you for helping keep Mintax and its users safe.
